## 设置签名（cat .git/config可以查看签名，系统级别比较麻烦，在~/.gitconfig里面）

形式:

用户名:xxx

邮箱:xxx.@qq.com

作用:区分不同开发人员的身份

辨析：这里的设置签名和登陆远程仓库（代码托管中心（码云，GitHub）的账号密码没有任何关系）

命令：

项目级别/仓库级别：仅在当前本地库范围有效

git config

git config user.name 名字

git config user.email 邮箱

系统用户级别（更大范围）：登陆当前系统的用户范围

git config --global

git config --global user.name 名字

git config --global user.email 邮箱

优先级：

就近原则：项目级别优先于系统用户级别，二者都有时采用项目级别的签名

二者都没有是不允许的

## 前进后退版本

基于索引值操作【推荐】git reset --hard 索引值，索引值就是git reflog 出来数据的第一个，例子，像9c619b2

reset其实有三个参数，--hard是其中的一个

使用^符号(只能后退，操作百度)

使用~符号(只能后退，操作百度)

## 分支管理 （branch）

什么是分支：在版本控制中，使用多条线同时推进任务

分支的好处：1.同时并行多个功能开发，提高开发效率

2.各个分支在开发过程中，如果某一个分支开发失败，不会对其他分支有任何影响。失败的分支删除重新开始即可。



查看所有分支：git branch -v

创建分支：git branch 名字

切换分支： git checkout 分支名

合并分支

1.切换到要受修改的分支上面

2.git merge 提供修改的分支名

合并出冲突需要手动合并，它会在文档中给你一些标识

冲突如何解决：1.编辑文件，删除特殊符号

2.把文件修改到满意的地步，保存退出

3.git add 文件名

4.git commit -m "日志信息" (不能加文件名)

## 克隆代码

git clone 你需要克隆的地址



克隆的效果1.完整把远程库下载到了本地

2.把别名也拿了过来

3.初始化了本地库

克隆下来的代码是没有提交权限的，你必须收到项目邀请，需要别人提供的邀请码，

就是普通的网址，进入接受邀请就可以了
